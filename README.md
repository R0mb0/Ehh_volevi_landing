<h2>🚀 Features</h2>
<ul>
<li><strong>Modern UI</strong>: clean, responsive layout with smooth gradients and shadows.</li>
<li><strong>Auto Theme</strong>: detects system light/dark mode automatically.</li>
<li><strong>Auto Language</strong>: switches between Italian and English based on browser language.</li>
<li><strong>Playful UX</strong>: “Try again” reload + meme animation; “Go back” can dodge the cursor.</li>
<li><strong>100% Static</strong>: pure HTML/CSS/JS, perfect for GitHub Pages.</li>
</ul>

<h2>🧩 Optional Troll Trick (GitHub Pages Redirect)</h2>
<p>
You can make the public link look legitimate by hosting a “serious” GitHub Pages site,
and then pointing its <code>index.html</code> to the "😏 Ehh Volevi?" page.
This way the URL looks official, but visitors get the troll page instead.
</p>

<h3>Example redirect <code>index.html</code></h3>

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta http-equiv="refresh" content="0; url=https://YOUR_USERNAME.github.io/MEME_REPO/" />
    <meta name="viewport" content="width=device-width,initial-scale=1" />
    <title>Redirecting…</title>
    <script>
      const REDIRECT_URL = "https://YOUR_USERNAME.github.io/MEME_REPO/";
      window.location.replace(REDIRECT_URL);
    </script>
  </head>
  <body></body>
</html>
```

<h2>⚙️ Setup</h2>
<ol>
<li>Clone or download the repository.</li>
<li>Open <code>index.html</code> to test locally.</li>
<li>Deploy with GitHub Pages (Settings → Pages → deploy from branch).</li>
</ol>
